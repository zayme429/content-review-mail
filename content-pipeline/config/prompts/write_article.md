# 文章结构要求

## 1. 引言（200-300字）
- **钩子**：用一个具体场景或数据引入，避免空泛
- **背景**：简要交代这个话题的重要性
- **问题**：提出一个核心问题或矛盾
- **预告**：简要说明文章将探讨的方向

## 2. 现象呈现（300-400字）
- **具体案例**：1-2个真实、有代表性的案例
- **数据支撑**：相关数据或研究结果
- **避免罗列**：不要堆砌多个浅层案例，要深入分析一个或两个

## 3. 深层分析（400-500字）
- **本质挖掘**：这个现象背后的深层逻辑是什么？
- **多维度视角**：技术、社会、心理、经济等角度
- **辩证思考**：既看到机会，也看到挑战
- **独特观点**：这是整篇文章的核心价值所在

## 4. 行动建议（300-400字）
- **可执行性**：具体、可操作，不是泛泛而谈
- **分层建议**：针对不同人群的差异化建议
- **资源推荐**：书籍、工具、课程、社区等（如有）
- **长期视角**：不只是短期应对，而是长期发展

## 5. 结语（100-200字）
- **回归主题**：呼应开头提出的问题
- **升华观点**：给出一个启发性的结论
- **开放性**：留给读者思考空间

# 写作要求

## 内容质量
1. **深度优先**：宁可少写几个点，也要把每个点写透
2. **案例鲜活**：案例必须有具体细节（人名、时间、场景）
3. **数据准确**：所有数据必须标注来源或说明是估算
4. **逻辑严密**：每个观点都要有推导过程

## 语言表达
1. **简洁有力**：删除所有冗余修饰词
2. **口语化但不口水**：像朋友聊天，但保持专业性
3. **避免术语堆砌**：必要的技术术语要解释
4. **节奏感**：长短句结合，避免大段文字

## 格式规范
1. **段落长度**：每段不超过4行（手机阅读友好）
2. **重点标注**：关键句用**粗体**标注
3. **小标题**：用### 层级清晰
4. **引用格式**：引用他人观点用> 引用格式
